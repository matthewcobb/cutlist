# v0.1.0

### Features

- **Demo project seeding** ‚Äî built-in demo import so new users can explore the app immediately (#1)
- **Part grouping** ‚Äî parts are grouped by dimensions and color; duplicate unplaced parts grouped by quantity instead of listed individually (#2, #8)
- **Part renaming & model preview** ‚Äî rename parts directly from the BOM; model thumbnail preview in the sidebar
- **Board layout improvements** ‚Äî better ruler rendering, improved pan/zoom controls, removed grain direction display on boards
- **BOM tab defaults** ‚Äî BOM tab now opens by default when viewing a project
- **"Updating" indicator in BOM** ‚Äî shows an "is updating" row while the packing engine recomputes layouts
- **Part form UX** ‚Äî dismiss part form on submit
- **Home button** ‚Äî navigate back to the landing page from any project
- **Part count warnings** ‚Äî surface warnings in the layout preview when part counts hit guardrails

### Performance

- **Web Worker packing** ‚Äî board layout generation runs off the main thread (#2)
- **Three.js optimizations** ‚Äî improved 3D viewer rendering performance
- **GPU memory leak fix** ‚Äî proper disposal of Three.js resources on teardown (#5)
- **Worker lifecycle cleanup** ‚Äî fix worker leaks and add part count guardrails (#5)
- **IDB write debouncing** ‚Äî debounce lightweight IndexedDB model writes to reduce write amplification (#5)
- **Raised part count limits** ‚Äî increased maximum part counts for larger projects

### Reliability & Data Layer

- **Production-hardened data layer** ‚Äî stricter IndexedDB schemas, versioned migrations, `applyDefaults` safety net, `FutureSchemaError` detection (#3)
- **Import validation** ‚Äî `.cutlist.gz` imports validated against strict Zod schemas; orphan model handling hardened
- **Centralized error reporting** ‚Äî `useAppErrors` composable for unified error handling (#10)
- **Sentry integration** ‚Äî client-side error monitoring with Sentry, including HMR guard and forwarding configuration
- **Page guard** ‚Äî route-level guard protecting against invalid project/tab navigation

### Code Quality & Testing

- **Test coverage** ‚Äî added tests for critical composables (`useModelHydration`, `useDemoSeed`, `useManualParts`) and packing engine performance regression tests (#4)
- **BOM refactor** ‚Äî extracted BOM row computation and filter logic out of `BomTab.vue` (#6)
- **Type tightening** ‚Äî named magic numbers, tightened types, fixed silent error swallows (#6)
- **Decomposed `useProjects`** ‚Äî split monolith composable into focused modules (`useModelHydration`, `useDemoSeed`, `useManualParts`) with tests (#7)

### Accessibility

- **ARIA attributes & keyboard support** ‚Äî added to interactive components (#6)
- **Stable keys & a11y fixes** ‚Äî fixed hardcoded colors, stable list keys for screen readers

### Infrastructure

- **CI** ‚Äî deploy only on release (not every push)
- **Environment config** ‚Äî `.env` setup, `.gitignore` updates
