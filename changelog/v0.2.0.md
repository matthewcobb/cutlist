# Changelog

## [v0.2.0] — 2026-04-24

_Since [v0.1.0] (2026-04-24)._

### Features

- **StripPacker** — new strip-based packing algorithm added to the tournament search alongside Shelf, Guillotine, and Tight packers.

### Code Quality

- **IDB layer on Dexie** — replaced the bespoke IndexedDB wrapper with [Dexie](https://dexie.org); schema versions and migrations are now declared declaratively in `useIdb/db.ts`.
- **Modular refactors** — split `exportPdf.ts` into `utils/pdf/` modules and `useIdb.ts` into a per-concern directory (`db`, `projects`, `models`, `buildSteps`, `layoutCache`, `demoSeed`).
- **Settings consolidation** — inlined `useProjectSettingsStore` into `useProjectSettings` and merged global settings into a single `useGlobalSettings` composable.
- **Packing engine cleanup** — deterministic search-pass budget, deduplicated placement helper, tightened packer types.
- **Test infrastructure** — centralised `fake-indexeddb` preload in `test-setup.ts` and expanded IDB test coverage.
- **Dead code removal** — dropped unused composables, utility functions, and a stored-only IDB field.

### Breaking Changes

- **Packing settings moved onto the project record** — schema bump; existing projects are migrated on load.
- **Settings excluded from project export/import** — `.cutlist.gz` files no longer carry a settings block; imports from older versions ignore the old field.

### Infrastructure

- **CI type-check on PRs** — `bun check` now runs on every pull request.
- **Dependency reorganisation** — dev vs. runtime dependencies tidied in `package.json`; `.gitignore` tightened to cover root `.nuxt`.
- **Docs** — README updated to match the actual package name and workflow triggers.
